    Section 6.1 Writing fractions in decimal

    Recall the idea of the Hindu–Arabic numeral system. The positions of the digits correspond to powers of \(10\text{,}\) and the digit in that place tells you how many copies of that power of \(10\) to include. For example,
    \begin{equation*} 4,071 = 4 \times 10^3 + 0 \times 10^2 + 7 \times 10^1 + 1 \times 10^0. \end{equation*}
    To expand upon this idea to represent non-whole numbers, rather than just having positions which represent repeated multiplication by \(10\) we want to also have positions which represent repeated division by \(10\text{.}\)
    🔗
    It is notationally convenient to use negative powers to represent repeated division. If \(-n\) is a negative number then
    \begin{equation*} b^{-n} = \frac{1}{b^n} \end{equation*}
    represents dividing by \(b\) \(n\) many times. Recall that the usual rules for exponents still work under this definition.
    🔗
    With this notation in hand, what we want is to expand our positional notation system to have positions not just for positive powers of \(10\) but also negative powers. The first thing you might try presents an issue. Suppose you want to write \(713\) to mean \(7 \times 10^1 + 1 \times 10^0 + 3 \times 10^{-1}\text{.}\) How do you know that you don’t mean \(7 \times 10^2 + 1 \times 10^1 + 3 \times 10^0\text{?}\) Or \(7 \times 10^{0} + 1 \times 10^{-1} + 3 \times 10^{-2}\text{?}\) When we only had positive powers there was no ambiguity—start with power \(0\) on the right. Now there is ambiguity.
    🔗
    The solution is simple: mark in some way where the zeroth power position is. The way we do this is via a decimal point, written as a period. Left of the decimal point is the positive power positions and right of the decimal point is the negative power positions. If only positive powers are included we don’t write the decimal point. Thus we can distinguish between the following.
    \begin{align*} 71.3 \amp= 7 \times 10^1 + 1 \times 10^0 + 3 \times 10^{-1}\\ 713 \amp= 7 \times 10^2 + 1 \times 10^1 + 3 \times 10^0 \\ 7.13 \amp= 7 \times 10^{0} + 1 \times 10^{-1} + 3 \times 10^{-2} \end{align*}
    🔗

    Definition 6.1.1. Hindu–Arabic numeral system, with fractional part.

    The Hindu–Arabic numeral system can be expanded to allow fractional parts as follows. Here, each \(d_i\) is a digit, a numeral \(0\) through \(9\text{.}\) First, an example with a fixed number of digits.
    \begin{align*} \amp d_2 d_1 d_0. d_{-1} d_{-2}\\ \amp= d_2 \times 10^2 + d_1 \times 10^1 + d_0 \times 10^0 + d_{-1} \times 10^{-1} + d_{-2} \times 10^{-2} \end{align*}
    And an example with an arbitrary number of digits, \(N\) before the decimal point and \(n\) after the decimal point.
    \begin{align*} \amp d_N \cdots d_1 d_0 . d_{-1} \cdots d_{-n}\\ \amp= d_N \times 10^N + \cdots + d_1 \times 10^1 + d_0 \times 10^0 + d_{-1} \times 10^{-1} + \cdots + d_{-n} \times 10^{-n} \end{align*}
    We call the part before the decimal point the integer part and the part after the fractional part.
    🔗
    🔗
    Any number written in this form can be written as a single fraction. For example, look at \(7.13\text{.}\) The fractional part represents the number of hundreths = \(\frac 1{100}\)s, so this number is \(7 + \frac{13}{100}\) or \(\frac{713}{100}\text{.}\) In general, if there are \(n\) digits after the decimal point then the number can be written as a fraction with denominator \(10^n\text{.}\)
    🔗
    Going in the other direction, any fraction whose denominator is a power of \(10\) can be written in this form. If the denominator is \(10^n\) then you need \(n\) digits past the decimal point. If there are not enough digits in the numerator you fill in the missing slots with \(0\)s. For example:
    \begin{align*} \frac{4513}{1000} \amp= 4.513\\ \frac{13}{1000} \amp= 0.013 \end{align*}
    🔗
    A reasonable question is, can any fraction be written in this form? If the fraction can be written to have a denominator which is a power of \(10\text{,}\) the answer is yes. That can happen just in case the denominator’s only prime factors are \(2\) or \(5\text{,}\) as those are the prime factors of \(10\text{.}\) For example:
    \begin{align*} \frac 12 = \frac 5{10} \amp= 0.5\\ \frac {31}{25} = \frac{124}{100} \amp= 1.24 \end{align*}
    For fractions with other denominators, however, it would take infinitely many digits past the decimal point to write exactly. For example, \(\frac13\) is written in decimal as \(0.33333\ldots\) where the \(3\)s repeat forever. The upshot of this is, the real numbers can’t just consist of numbers with finitely many digits past the decimal. For another example, the circle constant \(\pi\) requires infinitely many digits to write.
    🔗
    Those numbers which require infinitely many digits present complications for calculations. How do you add two numbers if there are infinitely many digits to look at? For practical purposes, however, this isn’t a problem. If you’re out in the real world you don’t know quantities to that level of precision. Thus it suffices to be able to work with finitely many digits. Our next topic is how to modify our algorithms for arithmetic to deal with these, and then we will talk about approximation.
